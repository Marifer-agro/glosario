GLOSARIO

Conceptos básicos de Git

Repositorio (repo)

Un repositorio es como una carpeta donde Git guarda todo el historial de cambios de tu proyecto. Puede estar en tu computadora (local) o en un servidor en línea (remoto) como GitHub.

Commit

Un commit es como una foto instantánea de tu proyecto en un momento específico. Cada vez que haces un commit, estás guardando el estado actual de tus archivos en el repositorio. Piensa en ello como un "guardar partida" en un videojuego.

Branch (rama)

Una rama es una línea de desarrollo separada dentro de tu repositorio. Puedes tener diferentes ramas para probar nuevas ideas sin afectar el código principal. Eventualmente, puedes fusionar las ramas (merge) cuando estés listo para integrar los cambios.

Github

¿Qué es Github?

GitHub es una plataforma en línea que actúa como una red social para desarrolladores, donde puedes alojar, compartir y colaborar en proyectos de software. Mientras que Git es la herramienta de control de versiones que utilizas localmente para rastrear los cambios en tu código, GitHub lleva esto un paso más allá al proporcionar un espacio donde esos cambios pueden ser almacenados y compartidos con otros de manera eficiente.

Repositorios en Github

Un repositorio en GitHub es donde almacenas todo el contenido relacionado con un proyecto: código, documentación, y recursos. Puedes tener repositorios públicos (visibles para todos) o privados (visibles solo para ti y los colaboradores que elijas).

Comandos básicos para sincronizar (Push, Pull, Fetch)

Git push

Una vez que has hecho cambios en tu repositorio local, puedes subirlos a GitHub utilizando este comando. Es como guardar la última versión de tu capítulo en la biblioteca para que todos los demás puedan leerlo.

Git pull

Este comando te permite actualizar tu repositorio local con los cambios que otros colaboradores han hecho en GitHub. Es como revisar si alguien ha agregado nuevo contenido al libro y asegurarte de que tienes la versión más reciente.

Git fetch

A diferencia de git pull, que actualiza directamente tu copia local con los cambios, git fetch solo descarga esos cambios para que puedas revisarlos antes de aplicarlos. Es como recibir una actualización sobre las nuevas secciones del libro sin integrarlas aún a tu copia.

HTML

HTML (HyperText Markup Language) es el lenguaje de marcado estándar utilizado para crear páginas web. A través de etiquetas y atributos, HTML define la estructura y el contenido de una página, permitiendo que los navegadores web muestren texto, imágenes, enlaces y otros elementos interactivos.
